# Phase quick Plan 260410-r60: Fix Regressions — Empty File Creation and Queue Panics Summary

**One-liner:** Eligibility-first guard in WriteLRC eliminates empty .lrc file creation; Next/Pop return errors instead of panicking on empty queue.

## What Was Done

Fixed two regressions introduced during package extraction:

1. **Issue #4 — WriteLRC empty file creation** (`internal/lyrics/writer.go`): `os.Create` was called before content eligibility was checked. If no synced, unsynced, or instrumental content was available the file was truncated/created empty on disk. Fixed by determining content type via a `switch` block before any file I/O, returning early with `"nothing to save"` error if none of the branches match.

2. **Issue #6 — InputsQueue panic on empty** (`internal/app/queue.go`): `Next()` and `Pop()` indexed `q.Queue[0]` without a bounds check, causing an index-out-of-range panic on an empty queue. Fixed by changing both signatures to `(models.Inputs, error)` with an `Empty()` guard, and updating all 4 call sites in `app.go` to handle the error return.
