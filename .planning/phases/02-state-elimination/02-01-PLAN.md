Create `internal/app/app.go` with the following:

**App struct** — owns all processing state:
```go
type App struct {
    inputs  *InputsQueue
    failed  *InputsQueue
    fetcher musixmatch.Fetcher
    writer  lyrics.Writer
    mode    string
    total   int
    cooldown int
}
```

**NewApp constructor** — accepts dependencies:
```go
func NewApp(fetcher musixmatch.Fetcher, writer lyrics.Writer, inputs *InputsQueue, cooldown int, mode string) *App
```
The constructor receives a pre-populated `inputs` queue (scanner fills it in main.go), creates a new `failed` queue internally, records `inputs.Len()` as `total`, and stores all fields. This keeps App decoupled from scanner and args parsing.

**Run(ctx context.Context) method** — the main processing loop:
- Iterates while `!a.inputs.Empty()`.
- On each iteration, **check `ctx.Err()` first** — if context is cancelled, break the loop immediately (this is how Ctrl+C stops processing).
- Call `a.fetcher.FindLyrics(cur.Track)`.
- On success, call `a.writer.WriteLRC(song, cur.Filename, cur.Outdir)`. Pop the item. If write fails, push to `a.failed`.
- On fetch error, pop and push to `a.failed`.
- Call `a.timer(ctx)` between iterations (only if items remain).
- After the loop exits (naturally or via cancellation), call `a.handleFailed()`.

**timer(ctx context.Context) method** — countdown with cancellation awareness:
- Instead of `time.Sleep` in a tight loop, use a `time.NewTicker(time.Second)` and `select` on both `ticker.C` and `ctx.Done()`.
- If ctx is cancelled during the timer, return immediately (don't print remaining countdown).
- Print `fmt.Printf("    Please wait... %ds    \r", remaining)` exactly as the original.
- Print `fmt.Printf("\n\n")` at completion.

**handleFailed() method** — replaces `failedHandler`:
- If inputs queue is not empty (interrupted mid-processing), append remaining inputs to failed queue: `a.failed.Queue = append(a.failed.Queue, a.inputs.Queue...)`.
- Log completion stats: `slog.Info("fetch complete", "success", a.total-a.failed.Len(), "total", a.total)`.
- If no failures, return nil.
- Log failed count.
- In "dir" mode: log "you can try again with the same command", return nil.
- In other modes: write failed items to `{timestamp}_failed.txt` using `bufio.Writer`, same format as current code (`artist,title\n` per line).
- Return error on file I/O failure instead of calling `os.Exit`.

**Important behavioral details to preserve:**
- The `handleFailed` method must handle the case where context was cancelled mid-loop — remaining unprocessed inputs go to failed list.
- The `fmt.Printf` calls for user-facing output stay as `fmt.Printf`, not slog.
- All slog.Info/Error calls match the current message strings exactly.
- Import `musixmatch` and `lyrics` packages for the interface types.
- Do NOT add `context.Context` to the `Fetcher` or `Writer` interfaces — that's v2 scope (CTX-01, CTX-02). The context is only used for loop/timer cancellation in App.

**Imports needed:** `context`, `bufio`, `fmt`, `log/slog`, `os`, `time`, plus internal packages `musixmatch` and `lyrics`.

Rewrite `main.go` to eliminate all global variables and use `signal.NotifyContext` for cancellation:

**Remove:**
- `var inputs = app.NewInputsQueue()` (line 30)
- `var failed = app.NewInputsQueue()` (line 31)

- `failedHandler()` function (lines 97-139)

**New main() function structure:**
```go
func main() {
    var args Args
    arg.MustParse(&args)

    // Create context with signal handling (replaces closeHandler goroutine)
    ctx, stop := signal.NotifyContext(context.Background(), os.Interrupt, syscall.SIGTERM)
    defer stop()

    // Build input queue (local variable, not global)
    inputs := app.NewInputsQueue()
    sc := scanner.NewScanner()
    mode, err := sc.ParseInput(args.Song, args.Outdir, args.Update, args.Depth, args.BFS, inputs)
    if err != nil {
        slog.Error("failed to parse input", "error", err)
        os.Exit(1)
    }
    fmt.Printf("\n%d lyrics to fetch\n\n", inputs.Len())

    if mode == "dir" {
        args.Outdir = ""
    } else {
        if err := os.MkdirAll(args.Outdir, 0750); err != nil {
            slog.Error("failed to create output directory", "error", err)
            os.Exit(1)
        }
    }

    // Resolve token (same precedence as before — Phase 3 will add env/dotenv)
    var token string
    if token = args.Token; args.Token == "" {
        token = "<HARDCODED_DEFAULT_TOKEN>"  // existing token from main.go, copy as-is
    }

    // Construct dependencies and run
    mx := musixmatch.NewClient(token)
    w := lyrics.NewLRCWriter()
    application := app.NewApp(mx, w, inputs, args.Cooldown, mode)

    if err := application.Run(ctx); err != nil {
        slog.Error("application error", "error", err)
        os.Exit(1)
    }
}
```

**Key changes from current main.go:**
- `inputs` is a local variable, not package-level.
- `failed` is not in main.go at all — it's internal to App.
- No `closeHandler` goroutine — `signal.NotifyContext` handles Ctrl+C.
- No `timer`, `failedHandler`, `closeHandler` functions — all moved to App.
- `Run` returns an error — main.go checks it.

**Imports to update:**
- Add: `context`, `os/signal` (already imported but now used differently)
- Keep: `fmt`, `log/slog`, `os`, `syscall`, `github.com/alexflint/go-arg`
- Keep: all internal package imports
- Remove: `bufio`, `time` (moved to app.go)

**Hardcoded token stays** — Phase 3 (API-02, API-03) handles externalization.
