# Quick Task 260411-l30 Plan 01: TOML Config + SQLite DB (Issues #11 and #12) Summary

**One-liner:** TOML config with XDG paths and env var overlay, plus pure-Go SQLite DB with embedded goose migrations, wired into main.go with --config flag and precedence-ordered token resolution.

## What Was Built

### Task 1: internal/config package + config.example.toml

- `internal/config/config.go` — `Config` struct with nested `APIConfig`, `OutputConfig`, `DBConfig`
- `Load(path string) (Config, error)` — reads TOML, falls back to XDG default path, overlays env vars
- `applyEnvOverrides()` — `MXLRC_API_TOKEN`, `MXLRC_COOLDOWN`, `MXLRC_OUTPUT_DIR`, `MXLRC_DB_PATH`
- XDG helper functions for `~/.config/<app>/` and `~/.local/share/<app>/` with Docker `/config` detection
- `config.example.toml` — documented example with all fields

### Task 2: internal/db package + initial migration

- `internal/db/db.go` — `Open(ctx, path) (*sql.DB, error)` with WAL, FK, busy_timeout=5000ms, synchronous=NORMAL
- Embedded migrations via `//go:embed migrations/*.sql` and `goose.UpContext()`
- `internal/db/migrations/001_initial.sql` — 5 tables: `libraries`, `scan_results`, `lyrics_cache`, `work_queue`, `api_keys`
- Partial unique index on `work_queue(artist, title) WHERE status='pending'`

### Task 3: main.go wiring

- `--config` flag added to `Args` struct
- Config loaded after `godotenv.Load()`, exits on error
- DB opened with signal context (ctx created before `db.Open()`), deferred close
- Token precedence: `--token` > `MUSIXMATCH_TOKEN` > `MXLRC_API_TOKEN` > config file
- Cooldown/outdir fall back to config values when CLI flags are at their defaults
