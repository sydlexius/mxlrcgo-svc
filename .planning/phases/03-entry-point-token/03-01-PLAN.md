Run `go get github.com/joho/godotenv@v1.5.1` to add the dependency to go.mod and go.sum. (Satisfies BUILD-07.)

**Step 2: Create `cmd/mxlrcgo-svc/main.go`.**
Create directory `cmd/mxlrcgo-svc/` and write `main.go` inside it. This file is a copy of the current root `main.go` with these modifications:

1. **Add godotenv import**: `"github.com/joho/godotenv"` in the import block.

2. **Add .env loading early in main()**: Right after `var args Args` / `arg.MustParse(&args)`, add:
   ```go
   // Load .env file if present (does NOT overwrite existing env vars).
   // Error is ignored -- .env file is optional.
   _ = godotenv.Load()
   ```
   `godotenv.Load()` reads `.env` from the current working directory and sets env vars that don't already exist. This is critical -- it means a real `MUSIXMATCH_TOKEN` env var takes precedence over the `.env` file value automatically.

3. **Replace the hardcoded token fallback** (lines 54-57 of current main.go). Replace the entire block:
   ```go
   var token string
   if token = args.Token; args.Token == "" {
       token = "<HARDCODED_DEFAULT_TOKEN>"  // the literal hardcoded token on this line
   }
   ```
   With token precedence chain (satisfies API-02, API-03):
   ```go
   token := args.Token
   if token == "" {
       token = os.Getenv("MUSIXMATCH_TOKEN")
   }
   if token == "" {
       slog.Error("no API token provided: use --token flag, MUSIXMATCH_TOKEN env var, or .env file")
       os.Exit(1)
   }
   ```
   This gives the correct precedence: CLI `--token` flag (checked first via args.Token) > `MUSIXMATCH_TOKEN` env var (which may have been loaded from .env by godotenv) > error if none found. The hardcoded default token is completely removed.

4. **Update the Token field help text** in the Args struct to document the precedence:
   ```go
   Token string `arg:"-t,--token" help:"musixmatch token (or set MUSIXMATCH_TOKEN env var / .env file)" default:""`
   ```

5. **Keep everything else identical** -- same imports (context, fmt, slog, os, signal, syscall, go-arg, internal packages), same Args struct fields, same signal.NotifyContext, same scanner/app wiring. The only changes are: add godotenv import, add `godotenv.Load()` call, replace hardcoded token with precedence chain, update Token help text.
